About Jems & Yen Bakery

Jems & Yen Bakery is a shop that focuses on selling bakery products established in 2021. Their speciality products are brownies and eclairs. Because the shop is still new, the owner wanted to create a website to reach new customers and increase people’s trust in the Jems & Yen Bakery brand.

Role & Responsibilites

Research & Design

Research

Project Brief

The owner of Jems & Yen Bakery and I had an online meeting. At that meeting, I asked a few questions regarding Jems & Yen Bakery and the website’s purpose.
The questions asked are as follows:

  • What does your business do ?
  • What’s the goal?
  • Who’s the target audience?
  • What’s the desired look & feel of the site?
  • Any specific functionality?

After getting information from the client, competitive analysis is carried out. After observing competitors’ websites, it was found that competitor made their websites with one of the features of Google. The websites have similarities with each other because they use templates.

Prototype

Wireframe

Before making a mockup, I will make a wireframe first so that when there is input or changes from the client, it will not take much time.

UI Style

Then proceed with determining the color, font, and icon used.

Review

Furthermore, the client provides feedback and input for the wireframe and selecting UI style. For the wireframe itself, there are changes where the image and tagline need to be changed while the UI style does not change.

Design

Hero Section

When the user opens the website, the first things that will be seen are pictures and information about the advantages of Jems & Yen bakery products so that visitors are more curious and move on to the next section.

About Us

In the about us section, visitors can see an explanation of jems & yen bakery.

Product

Then in our product section is information about the products being sold.

Location & Social Media

In the location section, there is the address of the jems & yen bakery shop. While underneath, there are contacts and social media.

Review

Next, the client checks the mockup results, whether there are things that are lacking or want to be changed. After the client is satisfied with the mockup, the website design process is complete.

Reflection

There are several things that can be learned in the making of this project.

  • It is very important to involve the client from start to finish.
  • Getting feedback from the client is important because if it doesn’t match his needs, a lot will have to change.

iPusnas

iPusnas is a free digital library application created by the National Library of the Republic of Indonesia. The iPusnas application has several features, such as borrowing books, reading books online and offline, and interacting with fellow users.

Before concluding the problem and providing solutions to the existing problems, I should do some research first to find out the experiences of other users.

Role & Responsibilites

User Research, prototyping, visual design & usability testing

Process

To make it easier to do research, I use design thinking.

Emphatize

Review Playstore

To learn about the issues that users are having with the iPusnas app, I looked at the reviews on the Google Play store. The findings of the review show that users have encountered problems:

  • Difficulty in registering an account
  • The navigation is messy
  • Ui display is very unattractive

Usability Testing

Usability testing is performed to learn more about the problems that users experience. Five participants did usability testing. Participants must be at least 18 years old and have used the iPusnas application.

There are 3 tasks performed in usability testing :

  1. Before being able to read the book, the user must register using e-mail.
  2. After creating an account, the user login.
  3. The user selects a book to read

Define

Affinity Diagram

Because of the huge amount of information collected during usability testing and interviews, an affinity diagram is required to organize and group it. Users confront some issues, including:

  • When you want to register, the user is confused because there is no register button.
  • The location of the navigation buttons is difficult to reach because they are above.
  • The interface is out of date.
  • Inconsistent product size.
  • The color of the button/icon is too dark, so it can’t be seen

Persona

Next, create a user persona based on the results of the research. By creating personas, we can empathize more deeply with the goals and problems of using the application.

Goals

  • Filling free time by reading books
  • Read books without any problems.

Frustation

  • Confusion when registering
  • The appearance of the application is not up to date
  • Difficulty accessing navigation buttons

User Journey

The next step is to create a user journey based on Budi’s user persona. With the user journey, I can understand the user experience when using the application and see opportunities that can be improved from the problems faced by the user.

Prioritization Idea

From the opportunities obtained from the user journey, the next step is to prioritize which opportunities must be done first. I prioritized changes to the registration flow, positioning of navigation buttons and changes to product size. Changing the registration flow is very important because we have to register to borrow a book. If the user cannot register, then he cannot borrow books. While the position of the navigation buttons and the size of the product because the user will often interact with it.

Ideate

Sitemap

For the sitemap itself, there are no changes or additions.

Userflow

Next, create a user flow that describes how a user performs a task. There are two user flows: “login/register” and “read a book”. The “login & register” flow’s registration has changed, however the “read a book” flow has not changed.

Login & Register

Read a book

Wireframe

Before making a mockup, I made a wireframe to get an idea of ​​how the design would look.

Prototype

Style Guide

For the colours themselves, I take the colours from the iPusnas application. At the same time, for typography, icons and spacing, use guidelines from material google because iPusnas does not provide guidelines or the type of typography used.

Margin & Spacing

For the margins, I went with 16 points, and for the spacing, I used multiples of 4, such as 8 and 16.

Mockup

Register Screen

On the registration screen, there is a change where there is an addition to the register button.

Home Screen

On the home screen, there are several changes, where the position of the home button that was previously on the top was moved to the bottom and consistent product size changes.

Book Detail Screen

On the book detail screen, there is a change where the image size is reduced then the position of the book description is on the top right so that the user does not need to scroll.

Test

Usability testing was required to validate the changes I made. Five people took part in the usability testing.

Participant Criteria

  1. Male / Female
  2. Age 18 years and over
  3. Understand using smartphones

Method

  1. Screen recording and observation notes
  2. Interview questions

Result

Based on usability testing results, all participants completed the given task.

Reflection

There are several things that can be learned in revising the iPusnas application

  • It is important that the login and registration features work properly.
